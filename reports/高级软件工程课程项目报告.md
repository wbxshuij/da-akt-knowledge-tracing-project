## 二、项目功能介绍

本项目是一个面向知识追踪任务的学生学习状态预测与诊断系统，主要用于根据学生历史答题记录预测学生未来答题表现，并进一步分析学生在不同知识点上的掌握情况。知识追踪是智慧教育中的重要任务之一，它通过分析学生与题目、知识点之间的历史交互序列，动态建模学生知识状态的变化过程，从而预测学生下一次答题是否正确。相比传统考试评价方式，知识追踪能够更加细粒度地刻画学生学习过程，因此适用于在线教育平台、智能题库系统、自适应学习系统和教师教学辅助场景。

本项目以 AKT（Attentive Knowledge Tracing）模型为基础进行改造。AKT 通过注意力机制建模学生历史答题行为，能够根据历史交互对当前预测的重要程度进行动态加权。为了使项目不只是简单复现原始 AKT，本项目进一步实现了 DA-AKT，即难度感知知识追踪模型。该模型在 AKT 主干结构基础上加入题目难度感知模块，根据训练集中每道题的历史正确率估计题目难度，并将题目难度作为额外特征融合到模型表示中，使模型能够区分同一知识点下不同题目的难易程度。

项目的输入数据主要包括学生编号、题目编号、知识点编号、答题结果和作答时间或序列顺序。其中，学生编号用于区分不同学习者，题目编号和知识点编号用于描述学生作答对象，答题结果用于表示学生是否回答正确，作答时间或序列顺序用于保证学生答题记录按照真实学习顺序排列。系统首先会对原始 CSV 数据进行字段检查、缺失值过滤、编号映射和序列构建，然后按照学生维度划分训练集、验证集和测试集，避免同一学生的数据同时出现在不同数据划分中。

在模型训练阶段，系统使用 DA-AKT 模型学习学生历史答题序列中的规律，并预测学生在未来题目上的答对概率。项目中特别注意知识追踪任务中的标签泄露问题。由于答题交互嵌入中包含学生当前答案，如果预测第 t 个位置时模型看到当前位置的答案，就会导致实验结果不可信。因此，本项目在注意力机制中采用严格历史交互协议，保证预测第 t 个答题结果时只能访问第 0 到第 t-1 个历史交互，而不能访问当前位置的答案信息。

从功能结构上看，本项目主要包括数据预处理模块、模型定义模块、训练评估模块、学习诊断模块、配置管理模块和实验报告生成模块。数据预处理模块负责清洗原始答题数据、构建学生序列并统计训练集题目难度；模型定义模块实现 DA-AKT 的嵌入层、难度融合层、因果注意力层、单调距离惩罚、残差连接、前馈网络和预测层；训练评估模块负责模型训练、验证集早停、最优模型保存以及 AUC、ACC、Loss 等指标计算；学习诊断模块负责生成学生层面的掌握度分析和知识点层面的风险分析；配置管理模块通过 YAML 文件统一管理数据路径、模型参数和训练参数；实验报告模块则在训练结束后自动生成 Markdown 格式的实验报告，方便后续整理课程作业。

本项目的实际价值在于，它不仅能够完成学生未来答题表现预测，还能够进一步输出学生学习诊断结果。例如，系统可以生成每个学生的平均预测掌握度、高风险答题比例和薄弱知识点列表，也可以统计每个知识点的真实正确率、模型预测掌握度和风险比例。这些结果可以帮助教师发现学生的薄弱知识点，为个性化练习推荐、学习路径规划和教学干预提供依据。

与原始 AKT 或 pyKT 通用实验框架相比，本项目的重点不是复现大量知识追踪模型，而是在 AKT 基础上完成一个更适合课程项目展示的工程化系统。项目增加了训练集统计型题目难度感知模块、严格防标签泄露的预测协议、学生诊断输出、知识点掌握度分析和自动实验报告生成。同时，项目还提供普通 AKT 消融配置，可以与 DA-AKT 进行对比，从而分析题目难度感知模块对预测效果和诊断结果的影响。整体来看，本项目形成了“数据处理—模型训练—模型评估—学习诊断—报告生成”的完整流程，具有较好的可运行性、可解释性和软件工程完整性。

## 三、本项目参考或基于的论文与项目来源

本项目主要参考了以下论文和开源项目思想：

1. AKT（Attentive Knowledge Tracing）模型。AKT 使用注意力机制建模学生历史答题交互，并引入单调距离惩罚思想，使模型在关注历史行为时能够考虑交互时间距离对当前预测的影响。
2. pyKT 知识追踪实验框架。pyKT 提供了多个知识追踪模型的统一数据处理、训练和评估思路，对本项目的工程组织具有参考意义。
3. 知识追踪任务的基本设定。知识追踪通常以学生历史答题序列为输入，输出学生对下一道题或当前知识点的答对概率，并使用 AUC、ACC 等指标进行评价。

需要说明的是，本项目并不是直接复制 pyKT，也不是将 pyKT 中的多个模型简单打包。本项目只选择 AKT 作为主干模型，并围绕课程项目要求进行了重新实现和工程化改造，重点突出“难度感知预测”和“学习诊断输出”。

## 四、本项目在参考项目基础上的改动

### 4.1 项目定位改动

原始 pyKT 更偏向知识追踪模型统一评测框架，而本项目定位为学生学习状态预测与诊断系统。因此，本项目不是追求包含尽可能多的知识追踪模型，而是围绕 AKT 主干模型构建一个完整可运行、可分析、可提交的课程项目。

### 4.2 题目难度感知模块

本项目在 AKT 基础上加入题目难度感知模块。系统首先只在训练集中统计每道题的历史正确率，然后计算题目难度：

```text
difficulty(q) = 1 - correct_rate_train(q)
```

难度越高，说明该题在训练数据中越容易被学生答错。为了方便神经网络建模，系统将连续难度值离散为多个难度区间，并使用难度嵌入表示。

### 4.3 难度门控融合机制

为了让模型自适应地决定题目难度信息的重要程度，本项目使用门控机制将难度嵌入与基础嵌入进行融合：

```text
gate = sigmoid(W[base_embedding; difficulty_embedding])
enhanced_embedding = base_embedding + gate * difficulty_embedding
```

这样模型既保留原始 AKT 的知识点表示和交互表示，又能够利用题目难度信息增强预测能力。

### 4.4 严格防标签泄露预测协议

知识追踪模型中，如果预测第 t 个答题结果时使用了包含当前答案 r_t 的交互嵌入，就会造成标签泄露。为了解决这一问题，本项目显式实现严格历史交互掩码：

```text
query t 只能访问 key < t
```

也就是说，预测当前位置时模型只能看见过去的历史答题记录，不能看见当前位置的答案。项目还提供了 `scripts/sanity_check_no_leakage.py` 脚本，通过翻转当前位置答案并观察当前位置预测是否变化来验证防泄露设计是否有效。

### 4.5 学习诊断输出

原始模型通常只输出 AUC、ACC 等预测指标。本项目在训练完成后进一步生成学习诊断结果，包括：

- `diagnosis_predictions.csv`：测试集中每条预测记录的学生、题目、知识点、难度、真实标签和预测概率；
- `concept_mastery.csv`：每个知识点的真实正确率、预测掌握度、平均风险分数和高风险比例；
- `student_diagnosis.csv`：每个学生的平均预测掌握度、高风险答题比例和薄弱知识点列表。

这些文件让项目从单纯的模型训练工具扩展为具有教学应用价值的学习诊断系统。

### 4.6 自动实验报告与消融实验

本项目训练结束后会自动生成 `experiment_report.md`，保存本次实验的关键指标、模型设置和输出文件说明。同时，项目提供 DA-AKT 和 Plain-AKT 两套配置，可以运行消融实验，对比是否加入题目难度模块对模型表现的影响。

## 五、系统设计与模块划分

项目采用模块化设计，核心目录如下：

| 模块 | 文件 | 作用 |
|---|---|---|
| 数据处理 | `src/da_akt_project/data.py` | 读取 CSV、字段检查、用户划分、序列构建、训练集难度统计 |
| 模型定义 | `src/da_akt_project/model.py` | 实现 DA-AKT 模型、严格因果注意力、难度门控融合 |
| 训练评估 | `src/da_akt_project/train.py` | 完成训练、验证、测试、早停、最优模型保存 |
| 诊断分析 | `src/da_akt_project/diagnosis.py` | 生成学生诊断、知识点掌握度和预测明细 |
| 工具函数 | `src/da_akt_project/utils.py` | 配置读取、随机种子、设备选择、JSON 保存 |
| 脚本工具 | `scripts/` | 生成样例数据、构建序列、防泄露检查、消融对比 |
| 配置管理 | `configs/` | 管理样例测试、真实数据实验和消融实验参数 |

系统整体流程如下：

```text
原始答题 CSV
    ↓
字段检查与数据清洗
    ↓
学生维度 train / valid / test 划分
    ↓
训练集题目难度统计
    ↓
序列数据构建
    ↓
DA-AKT 模型训练
    ↓
验证集选择最优模型
    ↓
测试集预测与 AUC / ACC 评估
    ↓
学生诊断、知识点分析、自动报告生成
```

运行 DA-AKT：

```bash
python -m src.da_akt_project.train --config configs/test.yaml
```

```bash
python -m src.da_akt_project.train --config configs/test_plain_akt_ablation.yaml
```

```bash
python scripts/compare_runs.py \
  --run_a outputs/test_run --name_a DA-AKT \
  --run_b outputs/test_plain_akt --name_b Plain-AKT \
  --output outputs/ablation_comparison.md
```

### 6.3 快速样例实验结果

压缩包内置样例数据主要用于验证项目完整流程是否可运行。快速测试结果如下：

| 模型 | 是否使用难度感知 | Best Valid AUC | Test AUC | Test ACC | Test Loss | 参数量 |
|---|---:|---:|---:|---:|---:|---:|
| DA-AKT | 是 | 0.5996 | 0.6335 | 0.5917 | 0.6765 | 103806 |
| Plain-AKT | 否 | 0.5484 | 0.5475 | 0.5281 | 0.7115 | 94846 |

从快速样例结果可以看出，加入题目难度感知模块后的 DA-AKT 在 Test AUC 和 Test ACC 上均高于关闭难度模块的 Plain-AKT。需要注意的是，样例数据规模较小，主要用于证明项目能够完整运行，正式课程报告中可以将 ASSIST2009 等真实数据集整理成相同 CSV 格式后重新运行，并替换为真实实验结果。

## 七、项目可靠性检查

### 7.1 防标签泄露检查

运行：

```bash
python scripts/sanity_check_no_leakage.py
```

脚本会构造同一条答题序列，并只翻转当前位置答案 r_t，然后比较当前位置预测是否发生变化。如果当前位置预测不变，说明模型预测当前位置时没有使用当前位置答案。项目检查结果为：

```text
PASS: strict previous-interaction protocol has no current-label leakage.
```

### 7.2 难度统计防泄露

本项目题目难度只从训练集统计，不使用验证集和测试集答案。对于测试集中出现但训练集中没有出现过的题目，系统使用训练集全局平均难度作为默认难度。这样既能保证模型可以处理未知题目，也避免了使用测试集标签构造特征。

### 7.3 工程可复现性

项目通过 YAML 配置文件固定数据路径、随机种子、序列长度、模型参数、训练轮数、学习率和 batch size。训练结束后保存 `used_config.json`，便于复现实验。项目还提供 `run.sh`、`run_ablation.sh` 和 `scripts/run_project_check.py`，方便一键检查完整流程。

## 八、项目总结

通过本次项目实践，我对知识追踪任务和深度学习项目的工程实现流程有了更加系统的理解。项目不仅涉及模型原理，还涉及数据清洗、序列构建、训练配置、模型评估、结果保存和报告生成等完整软件工程流程。在实现过程中，最主要的难点是如何让项目不只是简单复现 AKT，而是形成具有课程项目特色的完整系统。为了解决这个问题，本项目在 AKT 基础上加入题目难度感知模块，并进一步扩展学习诊断输出，使项目从单纯模型训练工具变成一个面向教学场景的学习分析系统。

在工程实践方面，本项目让我认识到项目结构和文档说明的重要性。清晰的目录划分、统一的配置文件、规范的数据格式和自动化输出，可以明显提高项目的可维护性和可复现性。通过 README 文件，用户可以快速了解项目功能、安装依赖、准备数据并运行实验。通过自动生成的指标文件和诊断文件，实验结果也更容易整理和分析。

本项目还让我认识到深度学习实验中数据协议的重要性。如果预测当前位置时模型看到了当前位置答案，即使代码可以运行，实验结果也会失去可信度。因此，本项目对注意力掩码进行了专门检查，采用严格历史交互协议，保证预测目标只依赖过去答题历史和当前题目信息。这个过程体现了软件工程中测试、验证和可靠性检查的重要性。

未来可以从两个方向继续完善本项目。一方面，可以在模型层面加入更多教育特征，例如作答时间、知识点先修关系、学生遗忘行为和题目文本信息，从而进一步提升预测准确率。另一方面，可以在工程层面增加 Web 可视化界面，将学生掌握度、薄弱知识点和预测结果以图表方式展示给教师和学生，使项目从离线实验代码进一步扩展为可使用的智能教育辅助系统。
